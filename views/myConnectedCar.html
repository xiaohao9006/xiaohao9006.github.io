Mongodb was set up on AWS server to collect data from Mojio Motion.

MongoDB Schema

There are three models in MongoDB - Trip, Vehicle and Location, and one trip has one vehicle and multiple locations.

Two independent Node JS servers were set up on AWS, one manages MongoDB and another handles REST API request.

Data Server

This server is responsible for pulling down my data from Mojio Motion platform, cleaning and making the data in the right structure, and then storing it in MongoDB.

REST API Server

This server is responsible for handling REST data request from client side. Vehicle statistics and trip analysis data is public, but since my detail trip info is a bit more private, so I added authentication to protect the route - users have to use LinkedIn account to login first before having access to this part of data.

Angular2 was used to organize my web app logic and wrap it as a single page application.

Application Structure

The great part of Angular2 is that it can help me modularize and componentize the application. Every component has its own MVC and won't conflict with each other. This web portal contains three modules - vehicle statistics, trip analysis and my all trips. Each module is associated with one navigation route, and has one to many components. Shared services are used to handle HTTP request and serve data to the application.

Angular2 Scaffolding Application Structure

Performance Optimization

Normally there are two factors that are going to slow down the web app, one is loading html, css and javascript to browser, and another is waiting for server to send back data.

Angular2 Preloading Strategy

Angular2 preloading strategy is used to optimize first issue - vehicle statistics module is the first view and is loaded at the very beginning, then almost immediately trip analysis and my all trips modules are loaded in the background. You can see three chunks of javascript are loaded in sequence (marked with redline) - inline.bundle.js contains vehicle statistics module, which is loaded at the very first, then after a while 0.chunk.js and 1.chunk.js are secretly loaded in background, which contains trip analysis module and my all trips module respectively.

Data Loading Animation

The second issue is inevitable - to display my last 30 days travel heatmap, server needs to send back around 60-70k geo coordinates, which usually takes about 2-3s. Therefore I created an interesting animation at the very beginning to indicate the data is loading.
